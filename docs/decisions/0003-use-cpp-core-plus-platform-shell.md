# 决策 0003: 使用 C++ 核心 + 平台外壳架构

## 状态

已接受

## 背景

UBAA Next 需要一个架构，要求：
- 最大化跨平台代码复用
- 将业务逻辑与 UI 分离
- 允许不同平台使用不同的 UI 框架

## 决策

使用 **C++ 核心 + 平台外壳** 架构：

- **C++ 核心**（UBAANextCore）— 所有业务逻辑、数据模型、抽象层
- **平台外壳** — 轻量 UI/入口层：
  - Windows CLI（当前 v0.4 稳定基线使用 mock/offline 验收并完成 CLI 工程化）
  - 通过 NAPI 连接 HarmonyOS ArkUI（v0.6/v0.7）
  - Windows Slint GUI（v0.8）

## 理由

1. C++ 核心可在所有平台间共享，无需修改
2. 平台外壳轻量，可独立替换
3. 基于接口的设计允许无平台依赖的 Mock 测试
4. 与原始 UBAA 架构一致（共享模块 + 平台 UI）

## 影响

- 核心不得依赖任何 UI 框架或平台 API
- 所有外部依赖通过抽象接口访问
- 每个平台外壳链接 UBAANextCore
- 提供 Mock 实现用于测试
