## 设计原则

1. **依赖倒置** — Core 依赖抽象（接口），而非具体实现
2. **单一职责** — 每个模块有且仅有一个明确目的
3. **接口隔离** — 小而聚焦的接口（IHttpClient、ISecureStore、ICacheStore）
4. **可测试性** — 所有业务逻辑可通过 mock 实现测试
5. **平台无关性** — Core 不包含任何平台相关代码

## 数据流

1. 平台 Shell（CLI/UI）接收用户输入
2. Shell 创建带注入依赖的相应服务实例
3. 服务调用模型层获取数据结构
4. 服务使用 IHttpClient/ISecureStore 进行外部操作
5. Result<T> 返回给 Shell 进行显示
