# 威胁模型

## 保护资产

- 学号、显示名等账号信息。
- 访问令牌、刷新令牌、CAS/业务系统 Cookie。
- 用户本地配置中的代理地址和连接模式。

## 信任边界

- 命令行参数来自用户输入，不视为可信数据。
- 校园 HTTP 响应来自外部系统，需要校验状态码、重定向和 Cookie。
- 本地文件系统可能被同机其他进程读取或篡改，因此真实会话文件需要平台加密。

## 已处理风险

| 风险 | 缓解措施 |
| --- | --- |
| 明文会话文件泄露 | Windows CLI 会话与 Cookie 文件使用 DPAPI 加密 |
| Cookie 响应头注入 | 拒绝 Cookie 名称、值、路径中的控制字符 |
| 过宽 Cookie 域 | 拒绝顶级域、空洞域名和非法域名字符 |
| 开放重定向 | 真实登录只跟随北航域名下的 HTTPS 重定向 |
| 命令注入 | CLI 不拼接 shell 命令执行用户输入 |
| 密码泄露 | 密码只在登录请求体内使用，不写入配置、会话文件或日志 |

## 剩余风险

- 非 Windows/OpenHarmony 当前没有真实平台安全存储实现，真实凭据持久化应由后续平台 Shell 接入系统密钥库。
- CLI 参数中的密码仍会出现在调用进程的命令行历史或进程列表中；后续可增加交互式密码读取。
- 当前 HTML 表单解析基于正则，足以覆盖现有 CAS 页面，但不是通用 HTML 解析器。
